---
name: resolving-merge-conflicts
description: "進行中の git マージ / リベースのコンフリクトを解決する必要があるときに使う。"
---

1. マージ / リベースの**現在の状態を把握する**。git の履歴と、コンフリクトしているファイルを確認する。

2. 各コンフリクトの**一次情報を探す**。それぞれの変更がなぜ行われたのか、元々の意図は何だったのかを深く理解する。コミットメッセージを読み、PR を確認し、元の Issue / チケットを確認する。

3. **各ハンクを解決する。** 可能な限り両方の意図を保つ。両立しない場合は、マージの目的に合う方を選び、そのトレードオフを記録する。新しい振る舞いを発明しては**いけない**。必ず解決すること。`--abort` は決して使わない。

4. プロジェクトの**自動チェックを探して実行する** — 典型的には型チェック、テスト、フォーマットの順。マージで壊れたものを修正する。

5. **マージ / リベースを完了させる。** すべてをステージしてコミットする。リベース中なら、すべてのコミットがリベースされるまでリベースを続行する。
