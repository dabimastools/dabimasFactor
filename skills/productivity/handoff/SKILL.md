argument-hint: "次のセッションは何に使う予定か？"

新しいエージェントが作業を継続できるよう、現在の会話を要約した引き継ぎドキュメントを書く。保存先は OS の一時ディレクトリ — 現在のワークスペースではない。

ドキュメントには「推奨スキル」セクションを含め、引き継いだエージェントが呼び出すべきスキルを提案する。

他の成果物（PRD、計画、ADR、Issue、コミット、diff）にすでに記録されている内容を重複させないこと。代わりにパスや URL で参照する。

API キー、パスワード、個人を特定できる情報などの機密情報は伏せ字にすること。

ユーザーが引数を渡した場合は、それを次のセッションの焦点の説明として扱い、ドキュメントをそれに合わせて調整する。
